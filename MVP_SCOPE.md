# MVP Scope — random-idea

## 1) Позиционирование
Для инди-разработчиков и вайбкодеров, которые сталкиваются с кризисом идей и зацикливанием на привычных шаблонах, мы создаём генератор стартап-идей из заранее продуманных смысловых блоков, который даёт быстрый поток нестандартных и пригодных к обсуждению концептов, в отличие от пустого листа, случайных советов в соцсетях и долгих брейнштормов без структуры.

## 2) Ценность MVP
- Получить 10-30 идей за одну короткую сессию.
- Сдвинуть мышление в новые комбинации ниш, ЦА, проблемы и моделей монетизации.
- Быстро отобрать 1-3 идеи для дальнейшей проверки.
- Сохранять удачные идеи в короткий список без лишних действий.

## 3) Ключевые гипотезы (для проверки)
- Пользователи видят ценность в том, что могут получить первую рабочую идею быстрее чем за 5 минут.
- Комбинаторный подход из смысловых блоков воспринимается как более полезный, чем полностью случайная генерация.
- Пользователи готовы возвращаться минимум 2 раза в неделю ради новых идей.
- Возможность сохранить и отобрать лучшие идеи повышает вероятность перехода к следующему шагу (исследование/интервью).
- Рекламная модель не ломает основной пользовательский поток при умеренной плотности показа.

## 4) Сегменты аудитории
- Core: инди-хакеры, соло-разработчики, вайбкодеры, которые регулярно запускают новые микро-проекты.
- Secondary: дизайнеры и продуктовые аналитики, которым нужны заготовки направлений для pet-проектов.
- Future: начинающие предприниматели без технического бэкграунда, ищущие форматную идею для первого запуска.

## 5) Границы MVP
### Включено в первую версию
- Генерация идей на основе набора смысловых блоков.
- Быстрый реролл идеи по кнопке.
- Сохранение идеи в персональный список.
- Отметка понравившихся идей и простой фильтр "избранное".
- Базовая страница с коротким объяснением ценности.
- Сбор базовой продуктовой аналитики по ключевым действиям.

### Не входит в MVP
- Персональные рекомендации на основе поведения пользователя.
- Автоматическая оценка рыночного потенциала идей.
- Коллаборативный режим с командным доступом.
- Глубокая кастомизация генератора по множеству параметров.
- Любые платные тарифы и сложная биллинг-логика.

## 6) Основные пользовательские сценарии
- Сценарий 1: пользователь заходит, нажимает "Сгенерировать", получает идею, повторяет цикл, сохраняет лучшие варианты.
- Сценарий 2: пользователь открывает список сохранённых идей и выбирает 1-3 для дальнейшей проверки.
- Сценарий 3: пользователь возвращается позже за новой сессией генерации.

## 7) Метрики успеха 30/60/90
### 30 дней
- Не менее 100 уникальных пользователей.
- Не менее 40% пользователей совершают 3+ генерации за первую сессию.
- Не менее 20% пользователей сохраняют хотя бы 1 идею.

### 60 дней
- Retention D7 не ниже 15%.
- Среднее число генераций на активного пользователя не ниже 8 за неделю.
- Не менее 30% активных пользователей сохраняют 3+ идеи.

### 90 дней
- Retention D30 не ниже 10%.
- Не менее 25% пользователей возвращаются как минимум 2 раза в месяц.
- Не менее 15% пользователей переходят к следующему шагу: формулируют и фиксируют выбранную идею для проверки.

## 8) Риски запуска и снижение
- Низкое качество идей: регулярно обновлять и расширять банк смысловых блоков, убрать слабые комбинации.
- "Игрушечное" восприятие продукта: добавить формат "идея + краткий контекст применения".
- Быстрый отток после первого визита: усилить сценарий возврата через "новые подборки недели".
- Перегруз рекламой: ограничить частоту показа, не прерывать критичный поток генерации.
- Нечёткая ЦА на старте: фокус на core-сегменте и отдельные посадочные формулировки под него.

## 9) Следующие артефакты после MVP Scope
- Problem Interview Script (10-15 интервью с core-аудиторией).
- JTBD-карта для сценария "хочу выбрать идею для следующего стартапа".
- MVP Metrics Sheet (события, воронка, retention, quality signals).
- Discovery Backlog с гипотезами для версии v1.1.
# MVP Scope: random-idea

## 1) Positioning statement
Для соло-фаундеров и инди-хакеров, которые сталкиваются с творческим тупиком при поиске идеи стартапа, мы создаём генератор неожиданных продуктовых идей в одну фразу, который даёт быстрый старт и снимает страх пустого листа, в отличие от ручного брейншторма, бесконечного скролла контента и однотипных списков идей.

## 2) Проверяемые гипотезы ценности (MVP)
1. Если пользователь получает идею за 10-20 секунд, он чаще возвращается за новой сессией генерации.
2. Неожиданность комбинаций повышает ощущение "нашёл что-то свежее" по сравнению с обычными шаблонными идеями.
3. Возможность сохранить удачную идею увеличивает долю пользователей, которые переходят к следующему шагу (оценка/проработка).
4. Простота интерфейса без лишних шагов повышает завершение первой сессии у новых пользователей.
5. Подборка "лучших за день" стимулирует социальное распространение и органический трафик.

## 3) Сегменты аудитории по приоритету
- Core: соло-фаундеры, инди-хакеры, вайбкодеры, которые регулярно ищут идею для нового микро-SaaS/продукта.
- Secondary: дизайнеры, маркетологи, аналитики и разработчики, которым нужны стартовые концепты для пет-проектов.
- Future: команды стартап-студий, образовательные программы, комьюнити-акселераторы и хакатоны.

## 4) Топ-5 рисков запуска и снижение риска
1. Риск низкой полезности идей (идеи кажутся случайным шумом).  
   Снижение: добавить фильтры по нише и типу пользователя, кнопку "ещё в этом же направлении".
2. Риск однотипных/повторяющихся идей.  
   Снижение: контроль повторов, расширение словаря компонентов, ротация шаблонов.
3. Риск низкого удержания после первого визита.  
   Снижение: сохранение избранного, история генераций, ежедневный триггер "новая порция идей".
4. Риск несоответствия рекламной модели ранней стадии.  
   Снижение: сначала фокус на росте и удержании, мягкая реклама только после подтверждения повторных сессий.
5. Риск неверного позиционирования для слишком широкой аудитории.  
   Снижение: в MVP фокус на core-сегменте и отдельные лендинговые формулировки по персонам.

## 5) Критерии успеха MVP (30/60/90 дней)
### 30 дней
- Запуск публичной версии с базовой генерацией идей.
- Не менее 100 уникальных пользователей.
- Не менее 40% пользователей генерируют более 3 идей за сессию.
- Сбор качественной обратной связи минимум от 20 пользователей core-сегмента.

### 60 дней
- Не менее 500 уникальных пользователей в месяц.
- Retention D7 не ниже 15%.
- Не менее 25% пользователей сохраняют хотя бы 1 идею.
- Определены 2-3 наиболее востребованных сценария использования.

### 90 дней
- Не менее 1500 уникальных пользователей в месяц.
- Retention D30 не ниже 8-10%.
- Не менее 30% трафика приходит органически (прямые заходы, шеры, рекомендации).
- Подтверждён repeatable use-case: у целевого сегмента есть регулярная потребность возвращаться.

## 6) Следующие артефакты для подготовки
1. Problem interview script для core-сегмента (цели, вопросы, критерии инсайтов).
2. JTBD-карта (functional, emotional, social jobs + triggers и desired outcomes).
3. MVP scope детализация (must-have / should-have / later).
4. Система метрик (acquisition, activation, retention, engagement, referral).
5. План тестов гипотез ценности (что проверяем, чем измеряем, пороги успеха).
6. Структура пользовательской обратной связи (шаблон интервью + форма быстрого фидбека).

## 7) Границы MVP (что входит / не входит)
### Входит
- Быстрая генерация идей в одну фразу.
- Базовый контроль качества: минимизация повторов в коротком окне.
- Сохранение удачных идей пользователем.
- Базовая сегментация/фильтры по направлению.
- Экран с историей недавних генераций.

### Не входит
- Глубокая автоматическая оценка рыночного потенциала идеи.
- Сложные коллаборативные функции для команд.
- Платные тарифы и сложная монетизация.
- Продвинутые персональные рекомендации на основе долгой истории.

## 8) Definition of Done для первой версии
- Пользователь без обучения понимает продукт и получает первую идею менее чем за 20 секунд.
- Пользователь может сгенерировать серию идей и сохранить лучшие без лишних шагов.
- Команда получает данные, достаточные для оценки гипотез ценности и решения о развитии MVP.
